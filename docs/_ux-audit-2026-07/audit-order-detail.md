## 0. สรุปสั้น

โครงหลักของหน้า **ดีแล้ว** (next-step banner เป็นจุดโฟกัสเดียว + แท็บ default ตามสถานะ + PERM ซ่อนเงิน) —
ปัญหาที่เหลือเป็น 4 กลุ่ม:

1. **การ์ดบิล 1,336 บรรทัดถูกยัดใน sidebar คอลัมน์ 1/3** ทั้งที่เป็นพื้นที่ทำงานเงินหลักของแอดมิน/บัญชี — และแท็บชื่อ "บิล/ไฟล์" กลับ**ไม่มีบิล** (คอมเมนต์ในโค้ดยอมรับเอง)
2. **ข้อมูลซ้ำ 3 จุด**: ที่อยู่จัดส่ง (sidebar ↔ แท็บจัดส่ง) · ยอดเงินรวม (สรุปราคา sidebar ↔ หัวการ์ดบิล) · ตรวจรับเสื้อลูกค้า (inline ใน overview ↔ การ์ดของเข้าในแท็บงานผลิต — **คนละกลไก คนละข้อมูล**)
3. **CTA แข่งกันเอง 2 จุด**: ปุ่มหลักบน PageHeader vs ปุ่มบนแถบขั้นต่อไป — ปุ่ม header ไม่เช็คด่านพร้อมผลิต กดแล้วไปตายที่ server ทั้งที่แถบข้างล่างบล็อกไว้ถูกแล้ว
4. **แท็บไม่อยู่ใน URL** — refresh/แชร์ลิงก์/กด back แล้วแท็บหาย · จุดน้ำเงินบนแท็บนับของไม่ครบ

Role: หน้านี้ควรยอมรับว่าเป็น **หน้าของแอดมิน/ฝ่ายขาย/เจ้าของ** — ช่างมีบ้านจริงที่ `/production/[id]` (เบสเคาะแยกโมดูลแล้ว, commit `dc4c765`) และกำลังจะได้จอโรงงานแยก ดังนั้นแท็บงานผลิตบนหน้านี้ = "สรุป+ลิงก์" พอ (ตอนนี้ก็เป็นแบบนั้นแล้ว — ถูกทางแล้ว อย่าถอย)

## 2. จุดรก / ปัญหา (อ้างโค้ดจริง)

### 2.1 การ์ดบิลอยู่ผิดที่ — งานเงินทั้งวงจรถูกบีบใน sidebar 1/3

- `order-billing-section.tsx` = component ใหญ่สุดของหน้า (1,336 บรรทัด): สร้างบิล 5 ชนิด + suggest + บันทึกชำระ + หัก ณ ที่จ่าย 3% + ใบเสร็จผูกงวด (tax point) + คืนเงิน + void + เตือนงวดไม่ออกใบกำกับ — ทั้งหมด render ใน `order-sidebar.tsx:348-355` คอลัมน์แคบ
- ขณะที่แท็บชื่อ "บิล/ไฟล์" มีแต่ไฟล์ — คอมเมนต์ยอมรับเองที่ `page.tsx:704`: `"── แท็บ บิล/ไฟล์: ไฟล์ 3 ชั้น (บิลอยู่ sidebar) ──"` และ `order-tabs.ts:39`: `"billing คืน null โดยตั้งใจ — บิลอยู่ sidebar (ไม่ใช่แท็บ)"` → ผู้ใช้กดแท็บ "บิล/ไฟล์" เพื่อหาบิลแล้วไม่เจอ = ป้ายหลอกโดยไม่ตั้งใจ
- **มือถือหนักสุด**: grid เป็น `grid-cols-1` (`page.tsx:621`) → sidebar ทั้งแผงไปต่อท้ายเนื้อหาแท็บ — การ์ดบิลกลายเป็น**ของท้ายสุดของหน้า** ใต้ลูกค้า/ข้อมูลออเดอร์/ที่อยู่/สรุปราคา ทั้งที่กติกา repo คือ mobile-first หน้า ops · แถบขั้นต่อไป "เรียกมัดจำก่อนเริ่มงาน → ไปที่การ์ดบิล" (`order-next-step.ts:80-85`) ก็พา scroll ดิ่งลงก้นหน้า

### 2.2 ข้อมูลซ้ำ — 3 คู่

### 2.3 CTA แข่งกัน — ปุ่มสถานะบน header vs แถบขั้นต่อไป

- ปุ่มหลักบน PageHeader = forward status ตัวแรก (`page.tsx:433,453-462`) — สถานะส่วนใหญ่มันคือ**ปุ่มเดียวกัน**กับที่แถบขั้นต่อไปโชว์อยู่ห่างกัน ~2 นิ้ว (เช่น INQUIRY: header "ยืนยันออเดอร์" + banner "ยืนยันออเดอร์")
- จุดต่างที่อันตราย: banner บล็อก `STATUS→PRODUCTION_QUEUE` เมื่อด่านพร้อมผลิตไม่ผ่าน แล้วโชว์ "ติดอะไร" (`order-next-step-banner.tsx:69-85`, `order-tabs.ts:58-68`) แต่**ปุ่ม header ไม่เช็ค readiness** — กดได้ → server ปฏิเสธ → เห็นแค่ toast error (`page.tsx:159-165`) ไม่เห็นรายการที่ติด · ขัดหลัก B8 ทางอ้อม (ปุ่มที่ระบบรู้อยู่แล้วว่าจะพัง)
- ปุ่ม CANCELLED และสถานะ rollback มีชั้น confirm ครบ (`page.tsx:350-425`) — ส่วนนั้นดีแล้ว

### 2.4 แท็บเป็น state ลอย — ไม่อยู่ใน URL

- `activeTab` เป็น `useState` (`page.tsx:109`) — refresh หาย · แชร์ลิงก์ "ดูใบส่งงานนี้" ไม่ได้ · กด back จากหน้า production กลับมาเจอแท็บ default ใหม่
- กระทบ notification/กระดิ่งในอนาคตด้วย — deep-link เข้า section ไม่ได้เลย

### 2.5 จุดน้ำเงินบนแท็บ นับของไม่ครบ

- `tabHasContent` (`page.tsx:326-336`): `docs` นับเฉพาะ attachments — แต่แท็บ docs โชว์ design versions (ชั้น 2) ด้วย (`order-files-card.tsx:35-38`) → มีแบบแต่ไม่มี attachment = ไม่มีจุด · `history` นับเฉพาะ `order.revisions` — ใบแก้ไข (CO) อยู่คนละ query (`order-change-orders.tsx:23`) ไม่ถูกนับ · `overview` เป็น `false` ตายตัว
- ผลคือจุดน้ำเงิน "เชื่อไม่ได้" — ผู้ใช้เรียนรู้แล้วจะเลิกดู

### 2.6 Sidebar "ข้อมูลออเดอร์" ยาวเกินการใช้จริง

- `order-sidebar.tsx:145-218` มีได้ถึง ~11 แถว: ที่ใช้ตัดสินใจรายวันคือ **กำหนดส่ง / ความเร่งด่วน / เทอมชำระ** — แต่แถวเหล่านี้จมอยู่ใต้ ประเภท/ช่องทาง/คนสร้าง/วันสร้าง/แก้ล่าสุด ซึ่งเป็น metadata อ่านนานๆครั้ง · กำหนดส่ง (ตัวเร่งงานทั้งโรงงาน) แสดงเป็นแถวจางเท่ากับ "วันที่สร้าง"

### 2.7 จุดเล็กที่เจอระหว่างไล่ (แถม — แก้ตอนแตะไฟล์นั้นพอ, boy-scout)

- `page.tsx:149` — query attachments ยิงทุกครั้งที่เปิดหน้า ทั้งที่ใช้แค่จุดบนแท็บ docs + เนื้อหาแท็บ docs (ยอมได้ ถ้าจะคงจุดน้ำเงิน)
- `page.tsx:269` vs `:272` — ระหว่าง `me` โหลด: `isSalesUp` default **true** (`!me ||`) แต่ `canSeeMoney` default **false** → เมนู ⋯ กระพริบโผล่ก่อนสิทธิ์จริงมา (ทิศ default ไม่ตรงกันเอง)
- `order-items-display.tsx` ใช้ตารางแนวนอน (`overflow-x-auto` `:198,342,383`) — DESIGN.md ข้อ Mobile บอกจอ < sm ห้ามใช้ scroll ตารางแนวนอนเป็นทางหลัก (หน้านี้ไม่ใช่ ops จ๋าแต่ช่างเปิดดูไซส์บนมือถือจริง)
- sidebar ใช้ `ui/section.tsx` ส่วนการ์ดฝั่งซ้ายใช้ `ui/card.tsx` — สอง surface คนละแบบบนหน้าเดียว (เบามาก ไม่เร่ง)

## 3. หน้าเดียวรับหลาย role — ควรจัดตาม "งานที่มาทำ" ยังไง

| Role | มาทำอะไรบนหน้านี้ | ที่ควรเห็นก่อน | หมายเหตุ |
|---|---|---|---|
| แอดมิน/ฝ่ายขาย | ตีราคา · ยืนยัน · เรียกมัดจำ/วางบิล · ตามสถานะ · สร้างใบส่ง | next-step + รายการ + **บิล** | ผู้ใช้หลักตัวจริงของหน้านี้ |
| เจ้าของ | สแกนสถานะ + ยอด/กำไร + ประวัติ | status bar + สรุปราคา/margin | อ่านอย่างเดียวเป็นหลัก |
| กราฟิก | อัปแบบ/ตามผลอนุมัติ | แท็บงานผลิต (การ์ดงานออกแบบ) | default tab ตอน DESIGNING พาไปถูกแล้ว (`order-tabs.ts:20-36`) |
| ช่างผลิต | ดูไซส์/จำนวน/ลาย | — | **บ้านจริงคือ /production/[id] + จอโรงงาน (งานถัดไปของเบส)** — หน้านี้ให้เขาแค่ "อ่านได้ ไม่หลง เงินไม่โผล่" ซึ่ง PERM ทำแล้ว (`showMoney` ตัดทั้งการ์ด ไม่โชว์ ฿0 — ถูกตามนโยบาย ⑦) |

ข้อสรุปเชิงจัด layout: **อย่าออกแบบหน้านี้เผื่อช่าง** — ลงทุนกับเส้นทางแอดมิน (เงิน+เอกสาร) กับเจ้าของ (สแกนเร็ว) แล้วปล่อยแท็บงานผลิตเป็น summary+ลิงก์แบบปัจจุบัน · จอช่างไปทำที่โปรเจกต์จอโรงงานแยก

ต่อยอด `142f7d2` แบบ surgical — ไม่รื้อ:

1. **แท็บใหม่ "เงิน/บิล"** (gate `canSeeMoney` — role ไม่เห็นเงิน = ไม่มีแท็บนี้เลย): ย้าย `OrderBillingSection` จาก sidebar มาเต็มคอลัมน์ 2/3 + ยกการ์ด "สรุปราคา+ต้นทุน/กำไร" (`order-sidebar.tsx:279-346`) มารวมไว้หัวแท็บ → ยอดซ้ำ 2 จุดยุบเหลือจุดเดียว · เปลี่ยนชื่อแท็บ docs → **"ไฟล์"** (ตรงของจริง)
2. **Sidebar เหลือ context บางๆ**: ลูกค้า + แถวสำคัญ (กำหนดส่ง/เร่งด่วน/เทอม — ดันขึ้นบน ทำเด่น) + metadata ที่เหลือพับใต้ "รายละเอียดเพิ่มเติม" + Marketplace · **ตัดการ์ดที่อยู่จัดส่งออก** (ความจริงอยู่ที่ใบส่งในแท็บจัดส่ง — ให้แท็บจัดส่งโชว์ที่อยู่ตั้งต้นจาก order.shipping* ตอนยังไม่มีใบส่งแทน) · สรุปราคาคงไว้ได้แบบ **ย่อ 1 บรรทัด** (ยอดรวม + ชำระแล้ว/ค้าง) กดแล้วเด้งไปแท็บเงิน — เจ้าของยังสแกนได้โดยไม่สลับแท็บ
3. **ปุ่มสถานะ header**: ตัดปุ่มหลักออก เหลือแถบขั้นต่อไปเป็น CTA เดียว (ทางเดินสถานะทั้งหมดยังครบใน dropdown ⋯ ตามเดิม `page.tsx:530-545`) — แก้ทั้ง CTA ซ้ำและช่อง readiness (2.3) ในดีดเดียว
4. **แท็บลง URL** (`?tab=` + `router.replace` shallow) + แก้ `tabHasContent` ให้นับ design/CO ครบ (2.5)
5. **ยุบตรวจรับซ้ำ**: ตัด `ReceiveTrackingInline` ออกจาก overview — เหลือใบตรวจรับ (goods receipt) ทางเดียว แล้วให้ badge "ลูกค้าส่งมา" ใน items ลิงก์ไปการ์ดของเข้า *(แตะ flow ข้อมูล — ต้องเช็คก่อนว่า `order.updateReceiveTracking` มีจอไหนอ่านค่า `receivedInspected` อยู่บ้าง แล้วเสนอเบสเคาะ)*

- **ได้**: งานเงินได้พื้นที่เต็ม + มือถือไม่ต้องดำดิ่งก้นหน้า · ป้ายแท็บตรงความจริง · CTA เดียว · ลิงก์แชร์ได้
- **เสีย**: แท็บเพิ่มเป็น 6 (SegmentedControl scroll แนวนอนบนมือถือ — มี `overflow-x-auto` รองรับแล้ว `page.tsx:614`) · anchor "billing" ของ next-step ต้องเปลี่ยนจาก scroll → สลับแท็บ (แก้ `order-tabs.ts:40-53` จุดเดียว มี pattern อยู่แล้ว) · เจ้าของเห็นกำไรช้าลง 1 คลิก ถ้าไม่ทำสรุปย่อในข้อ 2

### ทาง B — เลิกแท็บ กลับ single-scroll เรียงตาม workflow + การ์ดพับได้

เรียงการ์ดแนวตั้งตามลำดับงานจริง (รายการ → แบบ → ของเข้า → ผลิต → QC → ส่ง → บิล → ไฟล์ → ประวัติ) ทุกการ์ดพับได้ และ **auto-expand เฉพาะการ์ดของช่วงสถานะปัจจุบัน** (reuse ตรรกะ `defaultTabForStatus` เดิม)

- **ได้**: ไม่มีปัญหา "แก้รายการค้างแล้วสลับแท็บทิ้งฟอร์ม" (ตอนนี้ต้อง confirm ที่ `page.tsx:134-146`) · anchor ทุกจุดอยู่บนหน้าเสมอ ไม่ต้องสลับแท็บก่อน scroll (`pendingScrollRef` `page.tsx:111-124` หายไปทั้งก้อน) · Ctrl-F/พิมพ์หน้าได้ครบ
- **เสีย**: **เดินถอยหลัง** — หน้ายาวคือปัญหาที่เบสบ่นและแท็บเพิ่งแก้ไป (commit `142f7d2` เกิดจาก "การ์ด 9 ใบแยกเป็น 5 แท็บ" `page.tsx:613`) · ทุก section mount พร้อมกัน = query ยิงพร้อมกันหมด (billing/delivery/design/QC ต่างมี query ของตัวเอง) · state พับ/กางเป็นความซับซ้อนใหม่ · ขัด surgical

**คำแนะนำ: ทาง A** — ทาง B คือการยอมรับว่า redesign รอบก่อนผิดทาง ซึ่งหลักฐานไม่สนับสนุน (แท็บ+banner เพิ่งลงและตอบโจทย์ "ไม่รู้จะทำอะไรต่อ" ตรงๆ) · ปัญหาที่เหลือคือ**ของอยู่ผิดที่** ไม่ใช่โครงผิด

## 5. สิ่งที่ต้องคงไว้ (ห้ามแตะตอนลงมือ)

- **OrderNextStepBanner + readiness block** (`order-next-step.ts` ทั้งไฟล์ + `shouldGateOnReadiness`) — สูตร billingHandled ตรงกับ server แบบจงใจ (`order-tabs.ts:83-113`) ห้าม drift
- **defaultTabForStatus** — เปิดมาเจองานที่ต้องทำเลย เป็นหัวใจของ redesign รอบก่อน
- **PERM pattern ทั้งหมด**: `showMoney` ตัดการ์ด/คอลัมน์ทิ้งทั้งชิ้น (ไม่โชว์ ฿0) · `canPermsSetStatus` ซ่อนปุ่มที่ server จะปฏิเสธ · เมนู ⋯ ต้องมี "ใบสั่งงาน" ทุก role (`page.tsx:464-483` — review เคยจับ)
- **ชั้น confirm ของ CANCELLED / rollback / COMPLETED / SHIPPED** (`page.tsx:350-425`) รวมเช็คบิลค้างจากข้อมูลสด
- **accordion ต่อใบในการ์ดบิล** (`order-billing-section.tsx:490-517`) — ย้ายที่ได้ แต่โครงพับ/กางดีอยู่แล้ว
- **สถานะเดิน server-side เท่านั้น** — งานนี้เป็นงานย้าย layout ห้ามแตะ logic เปลี่ยนสถานะ
