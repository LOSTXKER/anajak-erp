### CC — ศูนย์บัญชาการผลิต `/production` (แกนกลาง — ทำก่อน UX ที่เหลือ) 【M】✅ 2026-07-08

`/production` เปลี่ยนจาก "3 โซน + toggle แท็บ/บอร์ด" → **ภาพรวมทั้งโรงงานหน้าเดียว**:
- **① ต้องรีบ** (บนสุด แดง): เลยกำหนด + มีปัญหา (step FAILED) + ติดด่านพร้อมผลิต — ยุบต่อออเดอร์ · ว่าง = ไม่โชว์
- **② สายการผลิต**: tile ต่อเลน (เลขงานใหญ่ + จุดแดงถ้าเลยกำหนด) + tile หลังผลิต (ตรวจ/แพ็ค/ส่ง) — แตะ tile = ลงลึกดูรายการการ์ดเดิม (LaneCardView + ปุ่ม/ผ่านรวด/ร้านนอกครบ)
- **③ คิวถัดไป**: รอเปิดใบผลิต + กดเปิดใบผลิตจากตรงนี้
- **④ งานของฉัน**: strip รอง — ขั้นที่คน login ถืออยู่ (ช่างไม่หลงในภาพรวม)
- **ตัดทิ้ง**: toggle แท็บเทคนิค/บอร์ดเลน (=เลิกบังคับเลือก view ก่อนเห็นงาน) · localStorage view pref
- **ใช้ซ้ำ 100%**: `production.kanban` + `user.me` เดิม (ไม่มี endpoint ใหม่) · LaneCardView / OrderCardHeader / dialog นับ QC-รับของ / handler ทั้งหมด · ไม่มีเงินบนจอ
- **verify**: tsc/lint/unit 495 · browser: เปิดเห็นภาพรวม · แตะเลนลงลึก · ปุ่มย้อน · console 0 error

## 2. แผนแบ่งงวด UX1–UX8 (เรียงตามเจ็บ×คุ้ม)

> ประมาณแรง: **S** = ครึ่งวัน · **M** = ~1 วัน · **L** = 1.5-2 วัน
> ทุกงวด commit แยก + verify ตามเกณฑ์ก่อนเคลมเสร็จ (type check ผ่าน ≠ ใช้งานได้)

### UX1 — จอช่าง ณ จุดงาน: ใบผลิตเห็นลาย+ไซส์ + ปิดขั้น 1-2 แตะ 【M】

หัวใจของ "จอทัช/มือถือช่าง" ที่เบสเคาะ — **ไม่สร้างแอปใหม่** ช่าง login คนจริงบน tablet ใช้หน้าเดิมที่เติมของให้ครบ (รักษาการลงทุน PERM: auto-claim + `changedBy` เป็นคนจริง)

**ขอบเขต**
- ขยาย `production.getById` (`production.ts:97-122`) select เพิ่ม: แบบ APPROVED ล่าสุด (thumbnail+ไฟล์+เลขเวอร์ชัน — pattern เดียวกับ job ticket) + variants ต่อไซส์ + ตำแหน่ง/ขนาดลายพิมพ์ — **ทุก field ที่เพิ่ม ไม่มีเงิน**
- การ์ดใหม่ "แบบ + ไซส์" บน `/production/[id]` เหนือรายการขั้นตอน: รูปลายแตะขยายเต็มจอ + ตารางไซส์ (จอเล็กเป็นการ์ดตาม DESIGN.md)
- ปุ่มเร็วบนแถวขั้นตอน (`production-steps-list.tsx`): ขั้นว่าง → ปุ่มใหญ่ "รับงานนี้" (ยิง updateStep เดิม — server auto-claim เอง) · ขั้นของฉัน → "เสร็จแล้ว" + ถ้าขั้นนับจำนวน เปิด bottom sheet ถามจำนวน (default = ที่เหลือ) = 1-2 แตะ · dialog เต็มเดิมย้ายไปหลังปุ่ม "เพิ่มเติม" (งานหัวหน้า: มอบงาน/หมายเหตุ)
- ~~ซ่อน dropdown "QC ผ่าน/ไม่ผ่าน" ต่อขั้นจากฟอร์มช่าง~~ — **ตัดออกจากขอบเขต** (เบสเคาะ §4 ข้อ 4: คงเดิม ช่างยังติ๊กเองได้)

**เกณฑ์เสร็จ (verify จริง)**: เปิด `/production/[id]` บนมือถือ — เห็นรูปลาย+เลขเวอร์ชัน+ตารางไซส์โดยไม่ออกจากหน้า · ปิดขั้นธรรมดา = 1 แตะ, ขั้นนับจำนวน = 2 แตะ · ช่างแตะงานคนอื่นไม่ได้เหมือนเดิม · ไม่มีตัวเลขเงินโผล่ที่ไหนบนหน้า
**ความเสี่ยง**: ต่ำ — เพิ่ม select+UI ไม่แตะ logic status · จุดระวังเดียว: แถวขั้นตอนแน่นขึ้นบนจอแคบ (ทดบนมือถือจริง)

### UX2 — จอพิมพ์เห็นลาย + ปิดปุ่มหลอก B8 【S-M】

**ขอบเขต**
- `getPrintQueue`/`listPrintRuns` (`services/print-run.ts`) select เพิ่ม design ล่าสุด (thumbnail, fileUrl, เลขเวอร์ชัน) → แถวคิว+รายการในรอบโชว์ thumbnail ~56px แตะเปิดไฟล์เต็ม + ป้าย "v3" กันพิมพ์ผิดเวอร์ชัน
- ปิดปุ่มหลอก: `print-runs/page.tsx` + `films/page.tsx` เช็ค `permAllows(me?.permissions, "manage_production")` ก่อนโชว์ปุ่ม mutation ทุกตัว (เปิดรอบ/พิมพ์จบ/ตัดแยก/ยกเลิก/หยิบใช้) — role อื่นเห็น read-only (pattern เดียวกับ PERM4 ~55 จุด)

**เกณฑ์เสร็จ**: ช่างพิมพ์เห็นลาย+เวอร์ชันทุกแถวโดยไม่สลับหน้า · login ด้วย role ขาย/บัญชี → ไม่เห็นปุ่ม mutation สักปุ่ม แต่ยังอ่านคิวได้
**ความเสี่ยง**: ต่ำมาก — select เพิ่ม + gate ปุ่มตาม pattern ที่มี

### UX3 — หน้าสร้างออเดอร์เบาลง (progressive disclosure) 【M】

เลือก **ทาง A** จาก audit: คงโครง 1-2-3 + sticky bar (เพิ่งรื้อ ผู้ใช้เพิ่งชิน) ผ่าเฉพาะจุดรก — ลด control ที่เห็นตอนเปิดหน้าจาก ~18-20 เหลือ ~8-9

**ขอบเขต**
- โซนราคาทั้งก้อน (fee + ภาษี + เงื่อนไขชำระ + สรุปราคา `page.tsx:528-589`) **render เมื่อ `hasItemContent` เท่านั้น** — สวิตช์มีแล้ว (`page.tsx:167`) เอามาคุม render (ห้ามแก้เงื่อนไขใน `itemHasContent`) → ฟอร์มว่างจอสั้นลงเกือบครึ่ง + กับดัก validation `page.tsx:272-283` หายเอง
- ย้าย "รูปอ้างอิงจากแชท" จากท้ายหน้า ขึ้นไปท้าย section 1 (จุด capture ตอนถือแชท)
- ยุบ "ส่วนเสริม" + "หมายเหตุรายการ" + "ค่าใช้จ่ายเพิ่มเติม" จาก empty-state dashed ก้อนใหญ่ → แถวปุ่มลิงก์ (+ ส่วนเสริม · + หมายเหตุ · + ค่าใช้จ่าย) เปิดเมื่อกด
- เก็บเบา: เลข section ให้ครบ · "คัดลอกลาย" เปลี่ยนจาก `<select>` ปลอมเป็นปุ่ม+dropdown จริง · toast เมื่อ priority ถูก auto-เปลี่ยน

**เกณฑ์เสร็จ**: เปิดหน้าใหม่ฟอร์มว่าง — ไม่เห็นช่องราคา/ภาษี/ส่วนลดสักช่อง · ใส่รายการแรก → โซนราคาโผล่ครบ ตัวเลขคำนวณเท่าเดิมเป๊ะ · เปิดเป็นใบสอบถาม (ลูกค้าอย่างเดียว) ยังได้เหมือนเดิม · draft ค้าง refresh แล้วไม่หาย
**ความเสี่ยง**: กลาง-ต่ำ — จุดระวัง: `itemHasContent` ใช้ร่วมกับระบบ draft (ใช้คุม render ได้ ห้ามแก้ข้างใน) · logic ราคา/validate ห้ามขยับทั้งก้อน (ดู §5)

### UX4 — ทีวีคิวรวม `/factory` 【M-L】 *(เคาะครบแล้ว — auth เบาลงเป็นบัญชีจอ ไม่แตะ schema)*

ของใหม่ชิ้นเดียวของทั้งแผน — จอ read-only ตอบ 4 คำถามจากระยะ 3-5 เมตร: เครื่องพิมพ์ทำรอบไหน · คิวถัดไป · งานไหนด่วน/เลยกำหนด · งานไหนติดปัญหา

**ขอบเขต**
- **Server ก่อน**: extract query pressQueue/packQueue (`task.ts:102-210`) + `packGateReady` (`task.ts:17`) ออกเป็น service `factory-board.ts` ให้ `task.myToday` กับ endpoint ใหม่เรียกตัวเดียวกัน (กัน drift — ห้าม copy-paste) · endpoint `factory.board` คืน `{activeRuns, printQueue, pressQueue, packQueue, problems, dueSoon, outsourceDue}` — **type ไม่มี field เงินโดยโครงสร้าง** · ถ้าโชว์ "ติดอะไร" ใช้เฉพาะ `waitingOn` (ห้ามหยิบ readiness detail ที่มีเลขบาท `production-readiness.ts:84`)
- **Auth (เบสเคาะ §4 ข้อ 1 — บัญชี login ค้างไว้)**: สร้างบัญชีผู้ใช้ "จอโรงงาน" role ช่าง + PERM override ตัดสิทธิ์เหลือน้อยสุด (อ่านอย่างเดียว) แล้ว login ทิ้งไว้บนทีวี — **ไม่มี model/schema ใหม่ ไม่มี middleware exclusion** (`/factory` อยู่หลัง auth ปกติ) · endpoint `factory.board` = protectedProcedure ธรรมดา · session หลุด = middleware เด้งหน้า login เอง (fail-closed โดยธรรมชาติ — จอโชว์หน้า login ให้หัวหน้า login กลับ ไม่ค้างจอเปล่า) · ปิดจอถาวร = ปิดใช้งานบัญชีที่ ตั้งค่า→ผู้ใช้ (กลไก setActive มีแล้ว) · trade-off ที่เบสรับแล้ว: session หมดอายุต้องมีคนหยิบคีย์บอร์ด login ใหม่
- **หน้า `/factory`**: นอก `(dashboard)` ไม่มี sidebar · ธีมมืด สีจาก token เดิม (ห้าม hex ใหม่) · ฐานตัวอักษร 28-32px เลขออเดอร์ 40-48px `tabular-nums` · layout 100vh ไม่ scroll ไม่มีปุ่ม · คอลัมน์: ปัญหา/เลยกำหนด(แดง) · DTF (รอบที่เดิน + คิวถัดไป 6-8 แถว "+N งาน") · รีด (x/y) · แพ็ค (ธง BLIND SHIP แดงใหญ่) + แถวล่าง "กำลังจะมา" · แถวงาน**โชว์ชื่อลูกค้า + ชื่อช่างที่ถืองาน** (เบสเคาะ §4 ข้อ 2-3)
- **ความสด**: `refetchInterval: 30_000` — เกิน 2 นาทีไม่มีข้อมูลใหม่ ขึ้นแถบเหลือง "ข้อมูลค้างตั้งแต่ HH:MM" (B8 ห้ามจอสวยแต่โกหก)

**เกณฑ์เสร็จ**: เปิดบนทีวีจริงในโรงงาน อ่านเลขออเดอร์ออกที่ 4 เมตร · payload ทั้งก้อนไม่มี field เงิน (ยืนยันด้วย type + grep) · ปิดใช้งานบัญชีจอ (setActive) แล้วจอเด้งออกจากระบบ · ถอดสาย LAN 2 นาทีแล้วแถบข้อมูลค้างขึ้น · `task.myToday` ผลเท่าเดิมเป๊ะหลัง extract service
**ความเสี่ยง**: กลาง-ต่ำ (ลดจากเดิม) — **ไม่เพิ่มผิว public เลย** (อยู่หลัง auth เดิม) · จุดระวังย้ายไปที่บัญชีจอ: ต้อง override สิทธิ์เหลือน้อยสุด + จอเปิดโล่งใครเดินถึงก็เห็นคิว (เบสรับแล้ว — เคาะให้โชว์ชื่อลูกค้าด้วย)

### UX5 — รายละเอียดออเดอร์ ชิ้นเล็กผลไว: ตัด CTA ซ้ำ + แท็บลง URL 【S】

**ขอบเขต**
- ตัดปุ่มสถานะหลักบน PageHeader (`page.tsx:433,453-462`) — เหลือแถบขั้นต่อไปเป็น CTA เดียว (ทางเดินสถานะยังครบใน dropdown ⋯ เดิม) → ปิดช่อง "ปุ่มที่ server จะปฏิเสธ" ในดีดเดียว
- แท็บลง URL (`?tab=` + `router.replace` shallow) — refresh/แชร์ลิงก์/back แล้วแท็บอยู่
- ซ่อมจุดน้ำเงิน `tabHasContent` (`page.tsx:326-336`): แท็บไฟล์นับ design versions ด้วย · แท็บประวัตินับใบแก้ไข (CO) ด้วย

**เกณฑ์เสร็จ**: ทุกสถานะ — ปุ่มบน header หาย แถบขั้นต่อไป+dropdown ⋯ ยังเดินสถานะได้ครบ · เปิด `/orders/x?tab=delivery` ตรงแท็บ · refresh ไม่เด้งแท็บ · มีแบบแต่ไม่มี attachment → แท็บไฟล์มีจุด
**ความเสี่ยง**: ต่ำมาก — layout + state ล้วน ไม่แตะ logic

### UX6 — รายละเอียดออเดอร์: แท็บ "เงิน/บิล" + รีด sidebar 【L】

**ขอบเขต**
- แท็บใหม่ **"เงิน/บิล"** (gate `canSeeMoney` — role ไม่เห็นเงิน = ไม่มีแท็บนี้เลย ตาม pattern PERM): ย้าย `OrderBillingSection` (1,336 บรรทัด) จาก sidebar มาเต็มคอลัมน์ 2/3 + ยกการ์ดสรุปราคา+กำไร (`order-sidebar.tsx:279-346`) มารวมหัวแท็บ → ยอดซ้ำยุบเหลือจุดเดียว · เปลี่ยนชื่อแท็บ "บิล/ไฟล์" → **"ไฟล์"** (ป้ายตรงของจริง)
- anchor "billing" ของแถบขั้นต่อไป เปลี่ยนจาก scroll → สลับแท็บ (แก้ `order-tabs.ts:40-53` จุดเดียว มี pattern แล้ว)
- รีด sidebar: ดันกำหนดส่ง/เร่งด่วน/เทอมขึ้นบนทำเด่น · metadata (คนสร้าง/วันสร้าง/แก้ล่าสุด ฯลฯ) พับใต้ "รายละเอียดเพิ่มเติม" · ตัดการ์ดที่อยู่จัดส่งออก (ความจริงอยู่ที่ใบส่งในแท็บจัดส่ง — ให้แท็บจัดส่งโชว์ที่อยู่ตั้งต้นตอนยังไม่มีใบส่งแทน) · สรุปราคาเหลือย่อ 1 บรรทัด (ยอดรวม+ชำระ/ค้าง) กดเด้งไปแท็บเงิน — เจ้าของยังสแกนได้ไม่เสียคลิก

**เกณฑ์เสร็จ**: มือถือ — เปิดออเดอร์ค้างมัดจำ กดแถบขั้นต่อไป "เรียกมัดจำ" → เด้งเข้าแท็บเงินทันที ไม่ต้อง scroll ก้นหน้า · role ช่าง → ไม่มีแท็บเงินในรายการแท็บ · สร้างบิล/บันทึกชำระ/void ทำงานเท่าเดิมทุกปุ่ม (test บนข้อมูล dev จริง) · แท็บเป็น 6 อัน scroll แนวนอนบนมือถือได้
**ความเสี่ยง**: กลาง — ย้าย component ใหญ่สุดของหน้า (ห้ามแตะ logic ข้างใน ย้าย layout ล้วน) · accordion ต่อใบในการ์ดบิลคงเดิม

### UX7 — ตารางสินค้าหน้าสร้างออเดอร์ → การ์ดมือถือ + matrix เป็น default 【L】

ชิ้นใหญ่สุดของฝั่งฟอร์ม — แยกจาก UX3 เพื่อให้ commit เล็ก verify ได้

**ขอบเขต**
- ตารางสินค้า 8 คอลัมน์: จอ < sm สลับเป็นการ์ด (pattern DESIGN.md มีตัวอย่างใน repo) · ย้าย "แพค" + "ส่วนลดต่อชิ้น" เข้าแถวขยาย "เพิ่มเติม" ต่อแถว (pattern เดียวกับ `print-table-row.tsx:122-173` ที่ทำไว้แล้ว) → เหลือ 5 คอลัมน์หลัก: แหล่ง·สินค้า·ราคา·จำนวน·รวม
- งานตัดเย็บ/ลูกค้าส่งมา: เปิด SizeMatrix เป็น default แทนช่องสี/ไซส์เดี่ยว (งานพวกนี้แทบไม่มีไซส์เดียว) — ช่องเดี่ยวคงไว้ให้ FROM_STOCK

**เกณฑ์เสร็จ**: เปิดบนมือถือจริง — ไม่มี scroll แนวนอนในฟอร์ม · กรอกงาน 40 ตัว 5 ไซส์ 2 สี ได้โดยไม่งง · **regression หน้าแก้รายการออเดอร์ด้วยเสมอ** (`OrderItemCard` ใช้ร่วมกับ `order-items-editor.tsx:23`) — แก้รายการออเดอร์เดิมแล้วยอด/variants ถูกต้อง
**ความเสี่ยง**: สูงสุดในแผน — component ใช้ร่วม 2 จอ + ผูก state matrix · ทำหลังงวดอื่นเก็บชัยชนะไปก่อนแล้ว

### UX8 — เก็บฝั่งผลิต: จุดรีด/แพ็ค + เศษเล็ก 【M】

**ขอบเขต**

| ทีวี: ความสด | **polling 30 วิ + แถบข้อมูลค้าง** — ไม่ทำ websocket | ทีม 5 คน over-engineer · B8 บังคับบอกเมื่อข้อมูลค้าง |
| ทีวี: การนำเสนอ | ตัวหนังสือใหญ่ ธีมมืด ไม่มีปุ่ม/ไม่ scroll/ไม่ marquee · ไม่โชว์รูปลายบนทีวี | อ่านที่ 4 เมตร · รูปติด auth `/api/files` และช่างดูลายบนจอทัชที่ login แล้ว |
| ปุ่มสถานะ header หน้า detail | **ตัดออก เหลือแถบขั้นต่อไป CTA เดียว** | ปุ่ม header ไม่เช็ค readiness — เป็นปุ่มที่ระบบรู้อยู่แล้วว่าจะพัง (ขัด B8) |
| Matrix หลายไซส์ | **default สำหรับงานตัดเย็บ/ลูกค้าส่งมา** | งานพวกนี้แทบไม่มีไซส์เดียว · ตัว matrix ทำงานดีอยู่แล้ว แค่ทางเข้าซ่อน |

## 5. สิ่งที่จงใจไม่ทำ (กัน scope บาน)

**ไม่ทำเลย**
- Wizard หลายขั้นหน้าสร้างออเดอร์ · single-scroll หน้ารายละเอียด (เหตุผล §3)
- display token + ตาราง schema ใหม่ให้จอทีวี — เบสเคาะใช้บัญชี login ค้างแทน (§4 ข้อ 1)
- แอป/route group ใหม่ให้จอทัช · websocket realtime · PIN ต่อ action บนจอทัช (friction จนช่างเลิกใช้ — บันทึกเป็น option ถ้าเกิดปัญหาจริง)
- refactor `LaneCardView`/`production/page.tsx` (949 บรรทัด) และ `order-billing-section.tsx` (1,336 บรรทัด) — ไม่มีใบงาน functional · กติกา surgical (UX6 ย้ายที่การ์ดบิล ไม่รื้อข้างใน)
- เงิน/ต้นทุนใดๆ บนทุกจอฝั่งผลิต (มติเบส 2026-06-12) · GL/job costing/auto-nesting/chat/designer (ตาม ROADMAP ข้อห้ามเดิม)
- marquee/auto-scroll บนทีวี — คิวล้นบ่อยค่อยพิจารณาสลับหน้าทุก 15 วิ เป็นงานหลัง

**ห้ามแตะระหว่างลงมือ (การ์ดกัน regress — AI session หน้าอ่านก่อนเริ่มทุกงวด)**
- logic ราคา client+server (`lib/pricing` · `order.ts:608-615`) — client เป็น preview, server คิดจริงเป็น Decimal · taxRate default 7 + สลับ marketplace (Gate B2)
- `itemHasContent` (`order-form.ts:268-281`) — ใช้คุม render ได้ ห้ามแก้เงื่อนไขข้างใน · กลไก draft localStorage 2 ชั้น
- `OrderNextStepBanner` + readiness + `defaultTabForStatus` (`order-tabs.ts`) — สูตร billingHandled ตรงกับ server แบบจงใจ ห้าม drift
- PERM pattern: `showMoney` ตัดการ์ด/คอลัมน์ทั้งชิ้น (ไม่โชว์ ฿0) · เมนู ⋯ ต้องมี "ใบสั่งงาน" ทุก role · ชั้น confirm CANCELLED/rollback/COMPLETED/SHIPPED
- `updateStep` + auto-claim + guards (`production.ts:395-539`) · จังหวะรอบพิมพ์ PRINTING→PRINTED→COMPLETED (กันฟิล์มสลับออเดอร์) · `evaluateHeatPressGate`/`packGateReady`
- `OrderItemCard` ใช้ร่วมกับหน้าแก้รายการ (`order-items-editor.tsx:23`) — แตะเมื่อไหร่ต้อง regression หน้าแก้รายการเสมอ
