  • ลำดับบังคับ (เบสเคาะ 12 มิ.ย.): "งานร้านนอกไปก่อนเสมอ" — ของที่เราคุมไม่ได้ต้องออกตัวก่อน งานในมือเรา (รีด DTF → แพ็ค) ทำท้ายสุด — ระบบเรียงคิวตามนี้เป็นค่าเริ่มต้นทุกงาน ช่างไม่เริ่มรีดทั้งที่เสื้อต้องไปร้านปักก่อน (ปรับต่องานได้ถ้าจำเป็น)
  • ส่งเสื้อเผื่อเสียต่อไซส์ — มาตรฐานสากลเผื่อ 2–5% ทุกไซส์ + แจ้งมูลค่าเสื้อให้ร้านรับทราบ (ไม่งั้นเคลมไม่ได้) — ใบส่งร้านมีช่องนี้ให้

  • ERP เป็นเจ้าของ "เหตุการณ์" — ยืนยันออเดอร์ / ใบเบิก / ใบคืน → ส่งคำสั่งไปสต๊อค
  • Stock เป็นเจ้าของ "ตัวเลขของจริง" — คงเหลือ / จองแล้ว / หยิบได้จริง — ที่เดียว ไม่มีเลขสองชุด
  • รหัสสินค้ารายไซส์-สี (SKU) ใช้ร่วมกันอยู่แล้ว เช่น S008-S001 = คอกลมครีม S — ไม่ต้องทำตารางแปลงเพิ่ม
  • เสื้อจากโรงเย็บ เดินผ่านใบสั่งซื้อ-ใบรับของของ Stock (โรงเย็บ = ผู้ขายรายหนึ่ง) — นับรับต่อไซส์ + ราคาทุนต่อตัวอัปเดตเองอัตโนมัติ
  • ฝ่ายผลิตใช้แอป Stock ทุกวันอยู่แล้ว (เบสยืนยัน) → ยอดจองต้องโชว์ในแอป Stock ที่เขาเปิดประจำ — คนเบิกเห็นว่าเสื้อล็อตไหนถูกจองให้ออเดอร์ไหน ไม่เบิกทับกัน
  • การสั่งซื้อ (หมึก/ฟิล์ม/เสื้อ/วัสดุ) อยู่ฝั่ง Stock ที่เดียว — มีวงจรครบอยู่แล้ว (ขอซื้อ→ใบสั่งซื้อ→รับของ→ทุนบันทึกเอง) ERP ไม่สร้างซ้ำ มีแค่ปุ่มลัด "ของไม่พอ → ขอซื้อ" เด้งข้ามไป
  • งานฝั่ง Stock ที่ต้องแก้: ประตูรับคำสั่ง (API) ให้ตัดยอดรายไซส์ได้ + เพิ่มระบบจอง + เปิดรับใบคืน — ฐานข้อมูลพร้อมแล้ว เป็นงานแก้จุดเดียวไม่ใช่รื้อ

6ต้นทุนต่อชิ้น — ตอบคำถามเบส "ซื้อเป็นล็อตแล้วคิดต่อชิ้นยากไหม?"

คำตอบสั้น: ไม่ยาก ถ้าคิดแบบที่โรงงานทั้งโลกใช้ — และระบบคำนวณให้อัตโนมัติได้โดยไม่มีใครต้องกรอกเลขหน้างานเลย (ไม่ขัดกับที่เบสเคาะไว้ว่าห้ามเพิ่มงานกรอก)

🚗 โรงงานอื่นเขาคิดแบบ "ค่าน้ำมันต่อกิโลเมตร": เติมน้ำมันเป็นถัง แต่รู้ว่ารถวิ่งกิโลละกี่บาท — ของเราคือซื้อหมึกเป็นขวด ฟิล์มเป็นม้วน ผงกาวเป็นถุง แต่รู้ว่าฟิล์มพิมพ์แล้ว 1 เมตรต้นทุนรวมกี่บาท (ฟิล์ม+หมึก+ผง+ค่าไฟ — วงการไทยอยู่ราว 25–50 บาท/เมตร เทียบราคาจ้างพิมพ์ข้างนอก 89–99 บาท/เมตร) — ตั้งเรตนี้ครั้งเดียว แล้วระบบคูณเองทุกออเดอร์
ก้อนต้นทุนระบบรู้ได้ยังไงใครต้องกรอก
👕 ตัวเสื้อ (ก้อนใหญ่สุด ~50–70%)ราคาทุนต่อตัวจาก Anajak Stock (อัปเดตเองทุกครั้งที่รับของเข้า)ไม่มีใคร — อัตโนมัติ
🎞️ ฟิล์ม+หมึก+ผง (~10–20%)ขนาดลาย × จำนวนตัว (กรอกตอนเปิดงานอยู่แล้ว) × เรตต่อเมตรไม่มีใคร — อัตโนมัติ
🏪 งานร้านนอกตามบิลร้าน (ถ้ากรอกตอนเปิดใบส่งร้าน — ไม่บังคับ)แอดมิน (ถ้าอยากเห็น)
👷 ค่าแรง + ค่าไฟ + ค่าเสื่อมเครื่องเหมาต่อชิ้น ตั้งครั้งเดียวใน Settingsไม่มีใคร — อัตโนมัติ

ผลที่เบสได้: เห็นกำไรขั้นต้นโดยประมาณของทุกออเดอร์ตั้งแต่ตอนตีราคา + รู้ว่างานแบบไหน/ลูกค้ารายไหนขาดทุน — แม่นพอสำหรับตั้งราคา (วัสดุพิมพ์เพี้ยน 20% กระทบต้นทุนรวมแค่ ~2–4% เพราะก้อนใหญ่คือเสื้อซึ่งใช้ราคาซื้อจริง) · ระบบเตือนเองเมื่อราคาซื้อล็อตใหม่แพงขึ้นเกิน 10% ว่า "อยากปรับเรตไหม"

⚖️ ไม่ขัดกับมติเดิม: ที่เบสเคาะไว้คือ "ห้ามบังคับกรอกต้นทุนหน้างาน" ซึ่งยังยืนเหมือนเดิม 100% — แบบนี้คือระบบคิดเองจากข้อมูลที่มีอยู่แล้ว ส่วนกำไรขาดทุนจริงทางบัญชียังคิดรายเดือนเหมือนเดิม อันนี้เป็นแค่ "เข็มทิศตอนตั้งราคา" · เปิด/ปิดได้ — รอเบสเคาะในคำถามท้ายเอกสาร

7ลูกค้า 3 กลุ่ม — เส้นทางเดียว ต่างกันที่ "ตัวเสริม"

ตามแบบระบบสากล: ไม่แยก flow ผลิตต่อกลุ่มลูกค้า (ทีมจะงง) — แยกที่ 4 ตัวเสริมรอบนอกแทน

กลุ่มตัวเสริมที่เปิดใช้
🏷️ แบรนด์เสื้อ (reseller)
สั่งซ้ำ มีลายเดิม		ราคาเฉพาะลูกค้า · ปุ่มสั่งซ้ำ (ดึงสเปกเดิมทั้งใบ — ลาย/ไซส์/เสื้อ/ตำแหน่ง) · blind ship · ของประกอบของแบรนด์ (ป้ายคอ/ถุงที่ส่งมาเอง) · เก็บ "สเปกการพิมพ์ที่เคยใช้จริง" ไว้กับลาย — สั่งรอบหน้าไม่ต้องถามใหม่
🏢 องค์กร/บริษัท
เอกสารครบ เครดิตเทอม		ใบเสนอ → PO → มัดจำ → ใบกำกับภาษีทุกงวด → ใบวางบิล → รองรับลูกค้าหักภาษี ณ ที่จ่าย 3% ตอนรับเงิน (อยู่ในแผนแล้ว — ไม่ทำ ยอดค้างปลอมทุกใบ) · จำนวนเป๊ะ = ธงพิเศษ (งานแจกพนักงานห้ามขาด)
💬 Custom รายชิ้น
จากแชท LINE		ทางลัดงานชิ้นเดียว: เปิดงาน–รับเงิน–ผลิต–ส่ง จบในจอเดียว ไม่ต้องเดินพิธีเท่างาน 300 ตัว · ปุ่มแนบไฟล์/สลิปที่ส่งตามหลังจากแชท

8จอเช้าของแต่ละคน — เปิดจอเดียวรู้งานทั้งวัน

กติกาเดียวกันทุกจอ: โชว์เฉพาะงานที่ลงมือได้ตอนนี้ งานที่ติดเงื่อนไข (รอไฟล์/รอเสื้อ/รอลูกค้า/ค้างร้าน) แยกกอง "ติดอะไร รอใคร" ต่างหาก

ช่างพิมพ์ DTF

🖨️ คิวพิมพ์ฟิล์ม

  • งานไฟล์พร้อม เรียงตามกำหนดส่ง
  • เลือกหลายงาน → รอบพิมพ์ → เสร็จทีเดียว
  • งานไฟล์ไม่พร้อม = ไม่โผล่ที่นี่
ช่างรีด

🔥 คิวรีด

  • เฉพาะงาน ฟิล์มเสร็จ ∧ เสื้อพร้อม
  • บันทึกคืบหน้าเป็นจำนวน (120/300)
  • สเปกตำแหน่ง/อุณหภูมิจากใบงานติดกอง
คนแพ็ค/ส่ง

📦 คิวแพ็ค

  • เฉพาะงานทุกสายเสร็จจริง
  • นับยืนยันต่อไซส์ + ใบรายการต่อกล่อง
  • ธง blind ship แดงเด่น
แอดมิน/ขาย

📋 ของเข้า–ออกวันนี้

  • ร้านนอกครบกำหนดรับวันนี้/เลยแล้ว
  • ของรอตรวจรับ · ลูกค้าค้างอนุมัติแบบ
  • งานครบกำหนดส่งวันนี้–พรุ่งนี้
เบส (เจ้าของ)

👁️ 5 ตัวเลข 10 วินาที

  • 🔴 งานเสี่ยงเลยกำหนด (กดเจาะดูได้)
  • ค้างร้านนอกกี่ใบ/เลยกำหนดรับกี่ใบ
  • คิววันนี้ เสร็จแล้ว/ทั้งหมด
  • บิลเลยกำหนด + ใบเสนอค้างตอบ
  • งานติดหล่ม (เงียบเกิน N วัน)
ทั้งโรงงาน

📺 จอกลาง (TV)

  • บอร์ดเลนเทคนิคที่ทำไปแล้ว ใช้ต่อ
  • ทุกคนเห็นภาพรวมเดียวกัน ไม่ต้องเดินถาม

9ของเดิมเก็บอะไร — รื้ออะไร — สร้างอะไรใหม่

เรื่องสถานะหมายเหตุ
เส้นเอกสาร-การเงิน (ใบเสนอ/มัดจำ/ใบกำกับ/วางบิล/วงเงิน)เก็บ — แข็งแรงสุดเพิ่มแค่ช่องหัก ณ ที่จ่าย 3% ตอนรับเงิน (แผนเดิมมีอยู่แล้ว)
เลนเทคนิค + ผ่านรวด + วงจรร้านนอก (เพิ่งทำ)เก็บเป็นฐานต่อยอดเป็นคิวต่อคน + เพิ่มจำนวน/กอง
ใบสั่งงาน Job Ticket + QR · ลิงก์ลูกค้าอนุมัติแบบ · แจ้งเตือนเก็บเพิ่มกติกา "แก้ออเดอร์แล้วใบเก่าตาย ต้องพิมพ์ใหม่"
ด่านพร้อมผลิต (เงิน/แบบ/ของครบ) + รายการเตรียมของสร้างใหม่หัวใจของรอบนี้
ใบตรวจรับของเข้า (นับต่อไซส์+รูป) + ใบคืนของลูกค้าสร้างใหม่กันข้อพิพาท — เคสเสื้อลูกค้าส่งมาคือเคสเสี่ยงสุด
รอบพิมพ์ฟิล์ม (รวมงาน) + คิวรีดแบบมีจำนวนสร้างใหม่ตรงวิธีทำงานจริงหน้าเครื่อง
นับยืนยันตอนแพ็ค + รายการต่อกล่อง + blind shipสร้างใหม่ลูกค้าแบรนด์ = กลุ่มรายได้หลัก
QC เชิงนับ (เสียกี่ตัว/สาเหตุ/รูป) + เผื่อเสียตอนเบิกสร้างใหม่ได้รายงานของเสียรายเดือนฟรี
คลังฟิล์มพร้อมรีด (พิมพ์เผื่อ — เบสยืนยันทำอยู่จริง)สร้างใหม่ลายไหน ลูกค้าไหน เหลือกี่ชิ้น — สั่งซ้ำรีดได้เลย
ต้นทุนต่อชิ้นอัตโนมัติ (เรตตั้งครั้งเดียว — ไม่มีงานกรอก)สร้างใหม่ (รอเคาะ)เห็นกำไรขั้นต้นตอนตีราคา — ดูหัวข้อ 6
ขั้นตอนผลิตผูกถึงระดับกอง/จำนวน (ไม่ใช่ทั้งออเดอร์)รื้อโครงฐานของทุกข้อบน — ทำก่อนเพื่อน
จองสต๊อค + เบิกผูกใบเบิก + ขาคืนของรื้อ+ต่อท่อ Stockฝั่ง Anajak Stock ต้องแก้ให้ตัดยอดรายไซส์ได้ (จุดเดียว)
แก้ออเดอร์กลางทาง (ใบแก้ไข + ผลกระทบ + คืน/เบิกเพิ่ม)รื้อปัจจุบันมีแค่ "ห้ามแก้" — ของจริงลูกค้าเปลี่ยนใจเสมอ
UI ทุกหน้า — โฉมใหม่ทั้งระบบรื้อ (เฟสท้าย)ออกแบบหน้าตาจริงหลัง flow ผ่าน — จะทำภาพตัวอย่างให้เบสเลือกก่อนลงมือ

10ลำดับการรื้อ (เสนอ 4 ก้อน — แต่ละก้อนจบแล้วใช้ได้เลย)

  1. กระดูกสันหลัง: ด่านพร้อมผลิต + เตรียมของ + เชื่อมสต๊อคจริง — รายการเตรียมของอัตโนมัติ · ใบตรวจรับ/ใบคืน · จอง-เบิก-คืนสต๊อครายไซส์ (แก้ประตู Stock + ระบบจอง) · ขั้นผลิตมีจำนวน (บอก "บางส่วน" ได้) — แก้ปัญหา "ของไม่ครบ/เดินถาม" ที่กินเวลาที่สุด
  2. หน้าเครื่อง: รอบพิมพ์ฟิล์ม + คลังฟิล์มพร้อมรีด + จอคิวต่อคน — คิวพิมพ์รวมรอบ+พิมพ์เผื่อ · จุดตัดแยกฟิล์ม · คิวรีดแบบ gate · จอเช้า 5 บทบาท + 5 ตัวเลขของเบส · ต้นทุนต่อชิ้นอัตโนมัติ (ถ้าเบสเคาะ)
  3. ขาออก: QC นับ + แพ็คยืนยัน + blind ship + แบ่งส่ง — ปิดวงให้ลูกค้าแบรนด์ใช้ได้เต็มรูป
  4. โฉมใหม่: UI ทั้งระบบ — ออกแบบ design ใหม่บนโครง flow ที่นิ่งแล้ว (ทำภาพให้เบสเลือกก่อน) + ทางลัดงานชิ้นเดียว + ใบแก้ไขออเดอร์
⚠️ ทำไมไม่รื้อทีเดียวทั้งหมด: ระบบมีของแข็งแรงอยู่ (การเงิน/เอกสาร/สถานะ) รื้อพร้อมกันหมด = โรงงานไม่มีระบบใช้หลายสัปดาห์และเสี่ยงพังย้อนหลัง — แบ่ง 4 ก้อนแต่ละก้อนจบในตัว เบสเห็นของจริงทุกก้อน ถ้าทิศผิดหยุดได้โดยไม่เสียของ

11มติทั้งหมด — เคาะครบแล้ว ✓ (12 มิ.ย. 2026)

  • รอบพิมพ์ฟิล์ม รวมหลายออเดอร์ + มีพิมพ์เผื่อ → ทำ "คลังฟิล์มพร้อมรีด"
  • ลำดับงาน: ทุกอย่างที่ outsource ส่งร้านก่อนเสมอ (คุมร้านนอกไม่ได้) — งานในมือเราทำท้ายสุด
  • เชื่อม Stock เป็นเนื้อเดียว: จอง-เบิก-คืนรายไซส์ · ยอดจองโชว์ในแอป Stock (ฝ่ายผลิตใช้ทุกวัน) · การสั่งซื้อทุกชนิดอยู่ Stock ที่เดียว ERP มีปุ่มลัด · เสื้อโรงเย็บรับเข้าเหมือนของซื้อ (นับต่อไซส์+ทุนอัตโนมัติ — ค่าเริ่มต้น ถ้าหน้างานติดค่อยปรับ)
  • ต้นทุนต่อชิ้นอัตโนมัติ: เอา — แบบตั้งเรตครั้งเดียว ไม่มีงานกรอกหน้างาน (เบส: "เอาก็ได้ ถ้าทำได้จริงและไม่งง")
  • blind ship มีจริง · ของแบรนด์ส่งมาแพ็คด้วยมีบ้าง → รองรับ
  • เผื่อเสีย: ค่าเริ่ม 3% แก้ได้ต่องาน · เหลือคืนเก็บเข้าสต๊อค
▶️ ขั้นต่อไป: เริ่มก้อนที่ 1 (ด่านพร้อมผลิต + เตรียมของ + เชื่อมสต๊อคจริง) — ใบงานละเอียดอยู่ใน ROADMAP.md ของ repo · งานแรกอยู่ฝั่งระบบ Stock (แก้ประตู API + ระบบจอง)
