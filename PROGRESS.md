- **คำสั่งหลัก**: `npm run dev` · `npm test` (unit — ต้องผ่านก่อน commit ที่แตะสูตร) · `npm run lint` (0 errors ห้ามถอย) · `npx prisma migrate dev` (ห้าม db push) · `npm run db:seed` (master data รันซ้ำได้) · `npm run verify:p02` (integration เงิน — สร้างข้อมูลจริง ห้ามรันบน DB ใช้งานจริง)
- **อ่าน `docs/ARCHITECTURE.md` ก่อนวางโค้ดใหม่** — ชั้น router(ผิว)/services(แกน)/lib + กฎเหล็กเงิน/สถานะ/เลขเอกสารอยู่ที่นั่นครบ
- **DB สะอาด ไม่มี order/customer** — เลขเอกสารเริ่ม 0001 เมื่อใช้จริง · ปัญหาเลขเก่าชน sequence หมดแล้ว
- **หนี้ที่จงใจค้าง + ใบงานรองรับ** (อย่าแก้เงียบ): lint warnings ~50 (react-hooks/no-img/unused) · platformFee → CostEntry/margin → P2 job costing · sidebar เมนูตาม role → รอบเก็บตก P1 · review เสริม 2 มิติ (pricing/decimal ละเอียด) ยังไม่ rerun
- **หนี้ใหม่จาก review งานมัดจำ/overdue + ใบวางบิล** (จดไว้ อย่าแก้เงียบ): (1) **ใบลดหนี้ไม่มีความหมายเชิงเงิน** — ไม่ผูกกับใบที่ลด/ไม่ลดยอดค้าง → จ่ายสุทธิหลังหักใบลดหนี้แล้วใบเดิมค้าง PARTIALLY_PAID → sweep mark OVERDUE ปลอม + ใบวางบิล/aging โชว์ยอดเต็ม (ตอนนี้เตือน amber ทั้ง dialog สร้างบิลและใบวางบิลแล้ว — ทางถาวร: เพิ่ม field ผูกใบที่ลด แล้วหักใน outstandingOf/เพดานจ่าย — ใบงานแยก มี test ก่อน) (2) แก้ออเดอร์ (update/updateItems/updateFees) ลดยอดต่ำกว่าบิลที่ออกแล้วได้โดยไม่เตือน — เพดานกันขาวางบิลขาเดียว (3) บันทึกชำระได้ทั้งบนใบแจ้งหนี้และใบเสร็จ → บันทึกซ้ำสองใบ totalSpent/รับชำระเดือนนับซ้ำ — แนวถาวร: auto-ออกใบเสร็จจากงวดรับเงิน (4) ปุ่มบันทึกชำระ/ยกเลิกบิลยังโชว์ role ที่ไม่มีสิทธิ์ (server กันอยู่) (5) ด่านวงเงิน SALES เป็น check นอก transaction — สอง request ยิงพร้อมกันลอดได้ (soft control โดย design: ผู้จัดการ override ได้อยู่แล้ว ยอมรับ) (6) เงินเข้าหลังใบวางบิลออก: recordPayment ไม่เตือนว่าใบอยู่บนใบวางบิล (หน้า list โชว์คงเหลือจริง+รับครบแล้วชดเชยแล้ว) (7) dropdown เลือกลูกค้าใน orders/new + quotations/new ตัดที่ limit 100 ไม่มีค้นหา — pattern เดียวกับที่เพิ่งแก้ในใบวางบิล ควรเก็บรอบ P1
- **หนี้จาก audit flow ทั้งระบบ (2026-06-11 — ช่องว่างที่หลุดแผน รอเคาะ/รอใบงาน):** (8) inquiry↔ใบเสนอ↔ออเดอร์ไม่ผูกกัน + convertToOrder ได้ items โครงเปล่า (OTHER/FREE ไม่มีลาย) — ควรเคาะ data model ตอนทำใบงาน "แก้ใบเสนอ" P1 (9) สถานะออเดอร์ไม่ sync เหตุการณ์จริง: ผลิตครบทุก step ออเดอร์ค้าง PRODUCING · delivery DELIVERED ไม่ขยับออเดอร์ · กด SHIPPED ได้โดยไม่มี tracking (10) global search บน topbar เป็นแค่เมนูลัด — พิมพ์เลขออเดอร์/ชื่อลูกค้าหาไม่เจอ (11) attachment.create มี API ไม่มีปุ่ม: ไฟล์ลูกค้าส่งตามหลัง/สลิปโอน (recordPayment รับ evidenceUrl แต่ dialog ไม่มีช่อง) (12) งานค้างสถานะนานไม่มีตัวเตือน (เช่น รอลูกค้าอนุมัติ 5 วัน — cron+กระดิ่งพร้อมแล้ว ขาดตัวกวาด) (12b) cosmetic: ออเดอร์ READY_MADE ที่ค้างสถานะ INQUIRY (เปิดงานเปล่าแล้วเติมเสื้อเปล่า → re-derive) — status bar หา INQUIRY ในเส้นทาง READY_MADE ไม่เจอ แถบ progress ไม่ไฮไลต์ขั้นปัจจุบัน (กดยืนยันออเดอร์แล้วหาย — การ์ดขั้นถัดไปชี้ทางอยู่แล้ว) (13) Delivery ไม่มี line items — แบ่งส่งหลายรอบบอกไม่ได้ว่ากล่องไหนมีอะไร ต้องเคาะ schema ก่อน P2 ใบแพ็ค · token อนุมัติแบบหมดอายุ regenerate ไม่ได้ · MaterialUsage เขียนได้อ่านกลับไม่ได้ (ผูกใบงาน P2 job costing) · ใบส่งของไปร้าน outsource ยังไม่มีเอกสาร (ถามเบสว่าหน้างานใช้อะไรอยู่)
